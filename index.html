El Idioma MarTech 🗣️

Conceptos básicos para hablar el mismo idioma técnico.

Explora los conceptos fundamentales de nuestra área. Este glosario interactivo está diseñado para alinear a todos los equipos sobre la terminología técnica que utilizamos a diario al hablar de analítica, marcaje y gestión de datos.

Página Virtual: Un evento que simula la carga de una nueva página en sitios tipo SPA (Single Page Applications) donde la URL no necesariamente cambia, pero el contenido principal sí.

Screen Name: El equivalente a una página pero en aplicaciones móviles (App). Representa la vista o pantalla actual en la que se encuentra el usuario. Se usa en Firebase/GA4 para apps nativas.
Es cualquier interacción del usuario con un elemento de la interfaz que no implica necesariamente un cambio de página o pantalla. Ejemplos comunes incluyen: hacer clic en un botón, enviar un formulario, abrir un menú desplegable, visualizar un popup o reproducir un video.

Estos eventos nos ayudan a entender la fricción y el uso funcional de la UI, y son fundamentales para calcular tasas de conversión y embudos.
El DataLayer (Capa de Datos) es un objeto JavaScript que almacena de forma estructurada información sobre la página o las interacciones del usuario. Funciona como un "puente" de comunicación seguro entre el sitio web y las herramientas de marketing (como GTM).

En lugar de raspar el HTML, GTM lee variables limpias directamente desde el DataLayer. Ejemplo básico:

window.dataLayer = window.dataLayer || [];
window.dataLayer.push({ event: 'click_boton', nombre_boton: 'Solicitar' });
Google Tag Manager es un sistema de gestión de etiquetas que permite actualizar de forma rápida y sencilla fragmentos de código (etiquetas) en un sitio web o app móvil sin tocar el código fuente constantemente.

Sirve como un contenedor central: nosotros inyectamos un solo código de GTM, y desde su interfaz gestionamos los píxeles de Facebook, eventos de GA4, scripts de VWO, tags de remarketing, etc. Esto da autonomía al equipo de MarTech sin depender de cada deploy de desarrollo.
GA4 es la versión más reciente de Google Analytics. A diferencia de Universal Analytics (UA), GA4 está basado en un modelo de eventos en lugar de sesiones, lo que permite una medición más flexible y precisa del comportamiento del usuario tanto en web como en app.

Ventajas clave: integración nativa con BigQuery, modelos predictivos con ML, medición cross-platform y cumplimiento con normativas de privacidad modernas.
Un Test A/B es un experimento controlado donde se muestra a diferentes grupos de usuarios dos versiones de una misma página o elemento (A = control, B = variante) para determinar cuál genera mejor resultado en una métrica objetivo (conversión, clics, tiempo en página).

VWO (Visual Website Optimizer) es la plataforma que usamos para crear, ejecutar y analizar estos experimentos. Permite definir el tráfico de cada variante, la métrica de éxito y calcular significancia estadística automáticamente.
